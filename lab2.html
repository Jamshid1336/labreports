Random Walk

random_walk(steps, step_length) is a function that makes the turtle walk a certain distance and then change to a random new direction and go the same distance again. This loops for as many times as the steps are defined to be. penup() and pendown() are just like drawing on a piece of paper. speed() makes the turtle walk faster or slower, so that the picture appears instantly or gradually. 



import random

# Step 1: Initialize the turtle canvas
canvas_width = 1000
canvas_height = 800
initializeTurtle(initial_window_size=(canvas_width, canvas_height), mode='auto')

# Step 2: Function to perform a random walk (Brownian motion)
def random_walk(steps, step_length):
    speed(4)             # Set speed to maximum for faster drawing
    penup()
    jump(450, 450)       # Start near the middle of the canvas
    pendown()

    for _ in range(steps):
        forward(step_length)               # Move forward by the step length
        right(random.uniform(-180, 180))   # Turn by a random angle between -180 and 180 degrees

# Step 3: Execute the random walk with increased steps and/or step length
steps = 25
step_length = 25

random_walk(steps, step_length)

# Step 4: Display the result
showturtle()
show()

Results & Analysis

When allowed to walk enough steps, the turtle will eventually leave the visible canvas.

Blip Line

Each repetition of the blip line for the next order creates 1/3 of the Koch Snowflake. blip_line() is the function defined that uses 1/3 of the length along with the order -1 to create more details. 



# Define the blip_line function
def blip_line(length, order):
    if order == 0:
        forward(length)
    else:
        blip_line(length / 3, order - 1)
        left(60)
        blip_line(length / 3, order - 1)
        right(120)
        blip_line(length / 3, order - 1)
        left(60)
        blip_line(length / 3, order - 1)

Koch Snowflake

Higher orders in the function provide more detail to the snowflake. Order 0 results in the original equilateral triangle that the snowflake stems from. 



initializeTurtle()
showturtle()

# Define the blip_line function (Koch curve)
def blip_line(length, order):
    if order == 0:
        forward(length)
    else:
        blip_line(length / 3, order - 1)
        left(60)
        blip_line(length / 3, order - 1)
        right(120)
        blip_line(length / 3, order - 1)
        left(60)
        blip_line(length / 3, order - 1)

# Draw the Koch snowflake
def koch_snowflake(length, order):
    for _ in range(3):
        blip_line(length, order)
        right(120)

The Dragon Curve

The Dragon Curve is self-repeating. It duplicates its full self with each increased order. In order 1, there are two sides making a right angle. In order 2, those two sides have doubled and swung away from each other, and it keeps repeating that operation. 



# Define the dragon curve function
def dragon_curve(order, length, turn_direction):
    # Base case: if order is 0, just move forward
    if order == 0:
        forward(length)
    else:
        # Calculate the new length for each recursive step
        new_length = length / (2 ** 0.5)

        # Recursively draw the dragon curve, turning right and left as needed
        dragon_curve(order - 1, new_length, 1)  # First segment
        right(turn_direction * 90)             # Turn by 90 degrees
        dragon_curve(order - 1, new_length, -1) # Second segment

# Initialize the turtle for drawing
initializeTurtle()

# Move the turtle to a starting position
jump(400, 400)
face(0)             # Set starting direction
color('purple')     # Choose a color

# Draw the dragon curve
order = 2          # Complexity level
length = 300        # Initial length of each segment
dragon_curve(order, length, 1)

# Show the final drawing
show()

Results & Analysis

It seems like with lower orders, it is easy to see the curve as made of boxes. When the order gets higher, it appears the viewer is either further away to view the full image, or the boxes are getting filled in as they replicate.
